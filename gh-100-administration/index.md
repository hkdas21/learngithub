# GitHub Administration — Exam Overview

## About This Certification

The **GitHub Administration** certification validates your skills in managing GitHub at scale. It covers the core competencies required to administer organizations, teams, security policies, compliance workflows, and enterprise features across the GitHub platform. Earning this certification demonstrates that you can effectively govern repositories, enforce access controls, integrate identity providers, and leverage GitHub Enterprise capabilities to support large development teams.

## Preparation Tips

1. **Hands-on practice is essential.** Create a test organization and experiment with every setting in the Organization and Enterprise dashboards. There is no substitute for real experience navigating the admin UI.

2. **Understand the permission model deeply.** Know the difference between base permissions, repository-level roles, custom roles, and team-level access. Be able to explain how these layers interact.

3. **Study identity and access management.** SAML SSO, SCIM provisioning, Enterprise Managed Users, and 2FA enforcement are heavily tested. Understand when each is used and how they differ.

4. **Learn branch protection rules and rulesets.** Know which settings require pull request reviews, enforce status checks, restrict force pushes, and lock branches. Understand how rulesets differ from classic branch protection.

5. **Review audit and compliance features.** The audit log, IP allow lists, webhook notifications, and security settings are critical for enterprise governance questions.

6. **Familiarize yourself with GitHub Apps vs. OAuth Apps.** Understand the permission model, installation scopes, and security differences between the two.

7. **Use the official GitHub documentation.** The GitHub Docs site is the authoritative source. Pay special attention to the "Organizations" and "Enterprise" sections.

8. **Take practice quizzes.** Use the quiz and flashcards in this study guide to test your recall before the exam.
