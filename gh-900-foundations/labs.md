## Hands-On Labs

Practice exercises to reinforce your learning. Complete each lab in order for the best experience.

---

### Lab 1: Create Your First Repository

**Objective:** Create a GitHub repository, add files, and make your first commit.

**Steps:**

1. Sign in to GitHub and click **New Repository**
2. Name it `my-first-repo`, set to Public, and check "Add a README file"
3. Click **Create repository**
4. Click the pencil icon on README.md and add a description
5. Write a commit message and commit directly to `main`

**Verification:** Your repository has at least 2 commits visible in the history.

---

### Lab 2: Branching and Pull Requests

**Objective:** Create a branch, make changes, and merge via a pull request.

**Steps:**

1. In your repository, click the branch dropdown and type `feature/add-about`
2. Click **Create branch: feature/add-about**
3. Create a new file called `ABOUT.md` with a short bio
4. Commit the file to the `feature/add-about` branch
5. Click **Compare & pull request**
6. Add a title and description, then click **Create pull request**
7. Review the changes and click **Merge pull request**

**Verification:** The `ABOUT.md` file appears on the `main` branch.

---

### Lab 3: Working with Issues

**Objective:** Create, label, and close issues using pull requests.

**Steps:**

1. Go to the **Issues** tab and click **New issue**
2. Title: "Add contributing guidelines", add the label `documentation`
3. Assign the issue to yourself
4. Create a branch, add a `CONTRIBUTING.md` file
5. Open a pull request and reference the issue with `Closes #1`
6. Merge the pull request

**Verification:** The issue is automatically closed after the merge.

---

### Lab 4: Forking and Contributing

**Objective:** Fork a public repository and submit a contribution.

**Steps:**

1. Find a repository with a `good-first-issue` label (try GitHub's own `github/docs`)
2. Click **Fork** to create your own copy
3. Make an edit in your fork
4. Click **Contribute > Open pull request**
5. Describe your changes and submit

**Verification:** Your pull request appears in the original repository.

---

### Lab 5: GitHub Projects

**Objective:** Set up a project board to track work.

**Steps:**

1. Go to your profile and click **Projects > New project**
2. Choose the **Board** layout
3. Add 3 columns: To Do, In Progress, Done
4. Create 3 draft items representing tasks
5. Convert one draft to an issue and move items across columns

**Verification:** Your project board shows items in different columns.
