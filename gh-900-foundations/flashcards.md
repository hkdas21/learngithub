## Key Terms & Concepts

Click each card to reveal the definition.

---

<details>
<summary><strong>Repository</strong></summary>

A container for a project that includes all files, revision history, and configuration. Can be public or private.
</details>

<details>
<summary><strong>Branch</strong></summary>

A parallel version of the repository. The default branch is usually `main`. Branches allow you to work on features without affecting the main codebase.
</details>

<details>
<summary><strong>Commit</strong></summary>

A snapshot of changes to one or more files, with a descriptive message. Each commit has a unique SHA hash.
</details>

<details>
<summary><strong>Pull Request (PR)</strong></summary>

A proposal to merge changes from one branch into another. Enables code review, discussion, and automated checks before merging.
</details>

<details>
<summary><strong>Fork</strong></summary>

A personal copy of another user's repository on GitHub. Forks let you make changes without affecting the original project.
</details>

<details>
<summary><strong>Clone</strong></summary>

A local copy of a remote repository, including all history and branches. Created with `git clone <url>`.
</details>

<details>
<summary><strong>Merge Conflict</strong></summary>

Occurs when two branches modify the same part of a file differently. Must be resolved manually before merging.
</details>

<details>
<summary><strong>GitHub Actions</strong></summary>

A CI/CD platform built into GitHub. Workflows are YAML files in `.github/workflows/` triggered by repository events.
</details>

<details>
<summary><strong>Issue</strong></summary>

A trackable unit of work — bugs, features, or tasks. Supports labels, assignees, milestones, and cross-references.
</details>

<details>
<summary><strong>CODEOWNERS</strong></summary>

A file that defines individuals or teams automatically requested for review when specific files are changed.
</details>

<details>
<summary><strong>Branch Protection Rules</strong></summary>

Settings that enforce workflows on specific branches — requiring PR reviews, status checks, or signed commits before merging.
</details>

<details>
<summary><strong>GitHub Codespaces</strong></summary>

Cloud-hosted development environments with a full VS Code experience. Configured via `devcontainer.json`.
</details>

<details>
<summary><strong>GitHub Projects</strong></summary>

A project management tool with board and table views, custom fields, and automation for tracking issues and pull requests.
</details>

<details>
<summary><strong>Two-Factor Authentication (2FA)</strong></summary>

An extra layer of security requiring a second form of verification (authenticator app, SMS, or security key) in addition to a password.
</details>

<details>
<summary><strong>Personal Access Token (PAT)</strong></summary>

An alternative to passwords for authenticating to GitHub via the API or command line. Supports fine-grained permissions.
</details>
