# GH-300: GitHub Copilot — Exam Overview

## About This Certification

The GitHub Copilot certification validates your skills in using GitHub Copilot effectively across the software development lifecycle. This exam covers prompt engineering, configuration, responsible AI practices, and enterprise management of GitHub Copilot. Earning this certification demonstrates that you can leverage AI-powered code assistance to improve developer productivity while adhering to best practices for responsible and secure usage.

## Exam Details

| Detail | Value |
|---|---|
| **Exam Code** | GH-300 |
| **Duration** | 120 minutes |
| **Number of Questions** | 75 |
| **Passing Score** | 70% |
| **Question Format** | Multiple choice / Multiple select |
| **Cost** | $99 USD |

## Exam Domains

| Domain | Weight |
|---|---|
| Responsible AI | 8% |
| GitHub Copilot Plans and Features | 16% |
| Prompt Crafting and Using GitHub Copilot | 28% |
| Developer Use Cases for GitHub Copilot | 22% |
| Testing with GitHub Copilot | 8% |
| GitHub Copilot Privacy and Data Considerations | 10% |
| Managing GitHub Copilot | 8% |

The largest domain is **Prompt Crafting and Using GitHub Copilot** at 28%, followed by **Developer Use Cases** at 22%. Together these two domains account for half the exam, so invest the majority of your study time in hands-on prompt engineering and real-world coding scenarios.

## Preparation Tips

1. **Get hands-on experience.** Install GitHub Copilot in VS Code or another supported IDE and use it daily. There is no substitute for real practice when it comes to prompt crafting and understanding suggestion behavior.

2. **Master prompt engineering.** Learn the difference between zero-shot and few-shot prompting, understand how the context window works, and practice writing clear, descriptive comments that guide Copilot toward accurate suggestions.

3. **Understand the plan tiers.** Know the differences between Copilot Individual, Business, and Enterprise — especially around policy controls, seat management, and data handling.

4. **Study responsible AI principles.** Be prepared to answer questions about AI ethics, bias awareness, the limitations of large language models, and when to trust or reject Copilot suggestions.

5. **Review privacy and data policies.** Understand how GitHub handles telemetry, code snippet retention, the public code filter, and organizational data policies.

6. **Practice test generation.** Use Copilot and Copilot Chat to generate unit tests, explore edge cases, and follow test-driven development workflows.

7. **Learn enterprise management.** Familiarize yourself with enabling Copilot for organizations, configuring policy settings, managing seats, reviewing usage analytics, and setting up content exclusions.

8. **Use official GitHub documentation.** The GitHub Docs site is the authoritative source for Copilot features, configuration, and administration.
